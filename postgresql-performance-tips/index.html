2. Index Foreign Keys

Every foreign key column should have an index. Without one, JOINs and cascading deletes do full table scans.

4. Tune shared_buffers

Set shared_buffers to 25% of available RAM. The default is far too low for production.

5. Enable Connection Pooling

Use PgBouncer or the built-in connection pooler. Opening a new connection per request is expensive.

6. Avoid SELECT *

Only fetch the columns you need. This reduces I/O and improves cache efficiency.

7. Use Batch Inserts

Instead of inserting rows one at a time, use multi-row inserts or COPY.

8. Vacuum Regularly

Postgres uses MVCC, which means dead rows accumulate. Autovacuum handles this, but monitor it.

9. Use JSONB Wisely

JSONB is powerful but not a replacement for proper schema design. Index JSONB fields you query frequently with GIN indexes.

10. Monitor with pg_stat_statements

Enable pg_stat_statements to see which queries consume the most time and resources.

Start with EXPLAIN ANALYZE — it will tell you where to focus your effort.
